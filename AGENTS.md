# AGENTS.md

这个文件用于告诉 AI 编程助手或协作者：修改本项目时应该遵守什么规则、项目目标是什么、哪些功能不要随意扩展。

## 项目定位

这是一个 Windows 桌面学习工具，不是大型学习平台。

核心功能：

- 用户导入单词。
- 用户手动添加单词。
- 本地记录学习状态。
- 轻量文字备忘录。
- 程序可隐藏到托盘。
- 支持 Windows 开机自启动。
- 支持可替换背景图。
- 支持背景可见模式和界面淡化程度调节。
- 支持独立背景预览页面。

开发时优先保持简单、稳定、容易理解。

## 技术原则

- 优先使用 Python 标准库。
- 不要引入 Web 后端。
- 不要引入复杂数据库服务。
- 不要默认依赖云服务。
- 不要要求用户登录。
- 不要把数据散落到多个系统目录。
- 默认数据存放在项目目录的 `data/` 文件夹内。

## 数据约定

项目数据目录：

```text
data/
  words.csv
  progress.json
  memo.txt
  settings.json
assets/
  background.png
```

约定：

- `words.csv` 是用户可编辑的单词源文件。
- `progress.json` 是程序维护的学习进度。
- `memo.txt` 是纯文本备忘录。
- `settings.json` 是程序设置。
- `assets/background.png` 是默认背景图。

不要把用户数据写死在代码里。
不要让背景图依赖用户个人目录。用户选择的新背景应复制到 `assets/` 中，并在设置里保存项目内相对路径。

## 功能边界

允许做：

- 单词导入。
- 单词手动添加。
- 单词列表展示。
- 标记认识或不认识。
- 保存和读取学习进度。
- 编辑和保存纯文本备忘录。
- Windows 自启动开关。
- 最小化到托盘。
- 更换和启用/关闭背景图。
- 启用/关闭背景可见模式。
- 调整界面淡化程度。
- 单独预览当前背景图。

暂时不要做：

- 番茄钟计时。
- 定时提醒。
- AI 生成内容。
- 云同步。
- 登录系统。
- PDF 或 Word 阅读器。
- 多用户系统。
- 复杂统计图表。

## UI 约定

- 界面应保持简洁。
- 主窗口只需要学习区、单词列表、备忘录、设置入口。
- 关闭窗口时优先隐藏，不直接退出。
- 退出程序必须通过明确的“退出”按钮或托盘菜单。
- 错误提示要让普通用户看得懂。

## Windows 兼容性

项目主要适配 Windows。

开机自启动应使用当前用户权限，不要求管理员权限。

可选方式：

- `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
- 当前用户启动文件夹

不要默认写入 `HKEY_LOCAL_MACHINE`，因为这通常需要管理员权限。

## 代码风格

- 保持单文件或少量文件结构，直到功能复杂度真的需要拆分。
- 函数名和变量名应清楚表达用途。
- 对文件读写、编码、路径处理要谨慎。
- 默认使用 UTF-8。
- 不要在异常处理中静默吞掉错误。
- 对用户数据写入前，应确保 `data/` 目录存在。
- 背景图相关逻辑应集中在独立函数里，避免散落在各个 UI 方法中。
- Tkinter/ttk 不支持单控件真实半透明；当前项目使用“背景可见模式 + 界面淡化程度”作为折中方案。不要使用整窗透明度淡化背景图。

## CustomTkinter 迁移约定

如果后续迁移到 CustomTkinter：

- 保留现有 `CSV`、`JSON`、`TXT` 文件存储逻辑。
- 保留 `assets/background.png` 和背景图复制逻辑。
- 优先只替换 UI 控件层，不重写业务逻辑。
- 单词表格可以暂时继续使用 `ttk.Treeview`，除非有明确替代方案。
- 不要因为迁移 UI 框架引入登录、云同步或数据库服务。

## 推荐开发顺序

1. 数据目录初始化。
2. 单词 CSV 读取。
3. 学习进度 JSON 读写。
4. 备忘录 TXT 读写。
5. 主窗口 UI。
6. 托盘行为。
7. Windows 自启动。

任何新增功能都应该先判断是否符合“轻量学习工具”的定位。
