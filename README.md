# Desktop Helper

一个适配 Windows 的轻量学习工具。它不是番茄钟，而是一个可以开机自启动、常驻托盘、随时展开的小型单词背诵和备忘录程序。

## 目标

这个项目的核心目标是简单、可控、方便管理：

- 开机后自动启动，默认缩小到系统托盘。
- 点击托盘图标后展开主页面。
- 支持用户自己导入单词。
- 支持极简文字备忘录。
- 数据保存在应用所在文件夹，方便备份、迁移和手动管理。
- 不做复杂提醒，不打断当前工作。

## 快速运行

开发运行：

```powershell
python main.py
```

如果希望启用系统托盘，需要先安装可选依赖：

```powershell
pip install -r requirements.txt
```

不安装依赖也能运行，只是关闭窗口时会最小化到任务栏，而不是隐藏到系统托盘。

## 使用说明

### 1. 启动程序

在项目目录中运行：

```powershell
python main.py
```

首次启动时，程序会自动创建 `data/` 文件夹，并生成示例单词、学习进度、备忘录和设置文件。

### 2. 添加或导入单词

如果只想添加少量单词，进入“单词列表”页面，点击“添加单词”。

弹窗中可以填写：

- `单词`：必填。
- `释义`：可选。
- `例句`：可选。
- `标签`：可选，例如 `cet4`、`work`、`daily`。

点击“保存”会写入 `data/words.csv` 并关闭弹窗。点击“保存并继续添加”会保留弹窗，方便连续录入。

如果添加的单词已经存在，程序会询问是否用当前填写内容更新已有单词，不会静默覆盖。

打开程序后进入“单词列表”页面，点击“导入 CSV”，选择你的单词文件。

CSV 文件建议使用下面的表头：

```csv
word,meaning,example,tag
apple,苹果,An apple a day keeps the doctor away.,basic
abandon,放弃,Do not abandon your plan.,cet4
```

字段说明：

- `word`：单词，必填。
- `meaning`：中文释义，可以为空。
- `example`：例句，可以为空。
- `tag`：标签，可以为空，例如 `cet4`、`work`、`daily`。

导入后，程序会把内容保存到 `data/words.csv`。

快捷键：

- `Ctrl+N`：打开“添加单词”弹窗。
- 在添加单词弹窗中，`Ctrl+S` 或 `Ctrl+Enter`：保存。

### 3. 学习单词

进入“今日学习”页面后，程序会按设置中的“每日学习数量”挑选未认识的单词。

你可以点击：

- “认识”：表示这个单词已经掌握。
- “不认识”：表示这个单词还需要继续看。
- “上一个 / 下一个”：手动切换当前单词。

学习状态会自动保存到 `data/progress.json`。

### 4. 查看和搜索单词

进入“单词列表”页面，可以查看：

- 全部单词。
- 中文释义。
- 标签。
- 当前学习状态。
- 上次学习日期。

顶部搜索框支持按单词、释义或标签筛选。

### 5. 使用备忘录

进入“备忘录”页面，直接输入少量文字，然后点击“保存备忘录”。

内容会保存到：

```text
data/memo.txt
```

备忘录是纯文本，不支持图片、附件或富文本格式。

### 6. 修改设置

进入“设置”页面，可以修改：

- 每日学习数量。
- 是否开机自启动。
- 是否启动后自动隐藏。
- 是否启用背景图。
- 是否启用背景可见模式。
- 界面淡化程度。
- 更换背景图。
- 恢复默认背景图。

开机自启动使用当前 Windows 用户权限，不需要管理员权限。

背景图会保存到项目的 `assets/` 文件夹中。更换背景时，程序会把你选择的图片复制到项目内，避免依赖原图片所在的个人目录。

如果背景被白色界面遮住，可以打开“背景可见模式”。这个模式会增加窗口边距、降低白色面板的厚重感，让背景露出更多。

也可以调整“界面淡化程度”。它只改变操作界面的面板观感，不会把背景图片一起淡化。

默认背景：

```text
assets/background.png
```

### 7. 托盘和隐藏窗口

如果已经安装托盘依赖：

```powershell
pip install -r requirements.txt
```

点击“隐藏窗口”或关闭窗口时，程序会隐藏到系统托盘。你可以通过托盘菜单重新显示窗口或退出程序。

如果没有安装托盘依赖，程序仍然可以运行，只是关闭窗口时会最小化到任务栏。

### 8. 管理和备份数据

点击顶部“打开数据文件夹”可以直接查看数据文件。

需要备份时，复制整个 `data/` 文件夹即可。

需要重置学习进度时，可以清空 `data/progress.json` 中的内容，或替换为：

```json
{
  "words": {}
}
```

### 9. 背景预览

进入“背景预览”页面，可以单独欣赏当前背景图。这个页面不会被学习区、单词表或备忘录的白色面板遮挡。

页面底部提供：

- 更换背景图。
- 恢复默认背景。
- 打开图片文件。

## 当前需求范围

第一版只做三个功能：

1. 单词学习
   - 用户可以手动添加或导入单词。
   - 每个单词包含英文、中文释义、例句、标签。
   - 支持标记为“认识”或“不认识”。
   - 可以查看今日学习列表和全部单词列表。

2. 备忘录
   - 只保存少量纯文本内容。
   - 不需要富文本、图片、附件或复杂分类。
   - 可以快速打开、编辑和保存。

3. 托盘与自启动
   - 程序启动后可以隐藏到系统托盘。
   - 点击托盘图标可以显示或隐藏主窗口。
   - 设置中可以打开或关闭 Windows 开机自启动。

## 暂不做的功能

为了保持项目简单，第一版暂时不做：

- 番茄钟倒计时。
- 单词定时提醒。
- 云同步。
- 登录账号。
- 多设备数据同步。
- AI 自动生成单词内容。
- PDF、Word 等复杂文档阅读器。
- 大型数据库服务，比如 MySQL、PostgreSQL。

这些功能可以以后再扩展，但不应该影响第一版的主流程。

## 推荐数据目录

数据建议直接放在应用所在文件夹下：

```text
desktop-helper/
  main.py
  README.md
  AGENTS.md
  assets/
    background.png
  data/
    words.csv
    progress.json
    memo.txt
    settings.json
```

说明：

- `words.csv`：用户维护的单词表。
- `progress.json`：程序记录每个单词的学习状态。
- `memo.txt`：备忘录文本。
- `settings.json`：程序设置，比如是否开机自启动、窗口位置等。
- `assets/background.png`：默认背景图。

注意：如果以后打包成 `.exe`，不要把程序放进 `C:\Program Files` 这类需要管理员权限的目录。更适合放在桌面、文档目录、D 盘或其他普通用户可写目录。

## 单词导入格式

第一版建议使用 CSV 文件，不直接上数据库。CSV 简单、可读、可用 Excel 编辑，也方便用户自己维护。

推荐格式：

```csv
word,meaning,example,tag
apple,苹果,An apple a day keeps the doctor away.,basic
abandon,放弃,Do not abandon your plan.,cet4
```

字段说明：

- `word`：单词。
- `meaning`：中文释义。
- `example`：例句，可以为空。
- `tag`：标签，比如 `basic`、`cet4`、`work`，可以为空。

程序启动时读取 `data/words.csv`，并把学习状态保存在 `data/progress.json`。这样用户可以随时修改单词表，而不会直接破坏学习记录。

## 为什么不先用数据库

这个项目的数据量很小，第一版不需要真正的数据库。

更推荐：

- 单词表：`CSV`
- 学习进度：`JSON`
- 备忘录：`TXT`
- 设置：`JSON`

如果以后单词量很大，或者需要搜索、筛选、历史统计，再考虑 SQLite。SQLite 也是一个本地文件数据库，不需要安装服务，Python 标准库自带 `sqlite3` 可以调用。

## 主界面建议

主窗口可以分成三个区域：

1. 今日学习
   - 显示今日要看的单词。
   - 提供“认识”和“不认识”按钮。
   - 显示今天已学习数量。

2. 单词列表
   - 展示全部单词。
   - 支持按标签筛选。
   - 支持重新加载 `words.csv`。

3. 备忘录
   - 一个简单文本框。
   - 自动保存或点击保存。

窗口行为：

- 点击关闭按钮时不退出程序，而是隐藏到托盘。
- 托盘菜单包含“显示主窗口”和“退出程序”。
- 真正退出程序时保存所有数据。

## Windows 开机自启动逻辑

推荐提供一个设置开关：

- 开启：在当前用户的启动项中注册本程序。
- 关闭：移除启动项。

Windows 上常见做法有两种：

1. 注册表：
   - 路径：`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run`
   - 优点：稳定，不需要管理员权限。

2. 启动文件夹快捷方式：
   - 路径：`shell:startup`
   - 优点：用户容易理解和手动删除。

第一版可以优先用注册表方式，但需要在设置界面明确显示当前状态。

## 后续开发顺序

推荐按这个顺序开发：

1. 先完成数据文件读写。
2. 再完成主窗口界面。
3. 再完成单词导入和学习状态。
4. 再完成备忘录保存。
5. 最后处理托盘和开机自启动。

不要一开始就做打包、AI、复杂 UI 或数据库，否则容易把简单工具做重。

## 技术建议

如果使用 Python，建议第一版使用：

- `tkinter`：做基础桌面窗口，Python 自带。
- `csv`：读取单词表，Python 自带。
- `json`：保存进度和设置，Python 自带。
- `pathlib`：处理路径，Python 自带。
- `winreg`：处理 Windows 注册表自启动，Python 自带。
- `Pillow`：处理背景图缩放和托盘图标，第三方依赖。

如果需要系统托盘，Python 标准库没有直接支持。可以后续考虑：

- `pystray`
- `Pillow`

本项目已经把托盘功能做成可选依赖：安装后启用，不安装也不影响核心学习功能。

## 背景图与 UI 精细化

当前版本使用 Tkinter/ttk，并已经把背景图逻辑集中在少量函数中：

- `copy_background_image()`：复制用户选择的背景图到 `assets/`。
- `create_cover_image()`：按窗口大小裁剪背景图，保持原图观感。
- `apply_background()`：把背景应用到主窗口。
- `choose_background_image()`：设置页里的“更换背景图”入口。
- `apply_visual_mode()`：切换背景可见模式和界面淡化程度。

Tkinter/ttk 不支持单个控件的真实半透明毛玻璃效果。当前版本提供的是可运行的折中方案，并且背景图不会被淡化：

- 背景可见模式：扩大边距并调整面板色彩，让背景更多露出。
- 界面淡化程度：只调整操作界面的面板颜色和遮挡感，不调整整窗透明度。

这种结构方便后续迁移到 CustomTkinter。迁移时可以保留现有的数据读写、单词逻辑、备忘录逻辑、自启动逻辑，只替换 UI 层。

推荐迁移顺序：

1. 安装 `customtkinter`。
2. 将 `Tk` 替换为 `customtkinter.CTk`。
3. 将 `ttk.Frame`、`ttk.Button`、`ttk.Entry`、`Text` 等逐步替换为 `CTkFrame`、`CTkButton`、`CTkEntry`、`CTkTextbox`。
4. 单词列表可以暂时保留 `ttk.Treeview`，因为 CustomTkinter 没有完全等价的表格控件。
5. 背景图继续复用 `assets/background.png` 和当前的背景处理函数。

可选依赖：

```powershell
pip install customtkinter
```

## 打包建议

后续如果想打包成 Windows `.exe`，可以使用 PyInstaller：

```powershell
pip install pyinstaller
pyinstaller --noconsole --onefile main.py
```

打包后建议把 exe 放在普通用户可写目录，例如桌面、文档目录或 D 盘目录。不要放到 `C:\Program Files`，否则 `data/` 数据文件写入可能需要管理员权限。
